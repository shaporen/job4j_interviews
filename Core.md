#### 1. Что такое виртуальная машина?

Виртуальная машина — это программная и/или аппаратная система, эмулирующая аппаратное обеспечение гостевой (target) платформы и исполняющая программы для target-платформы на host-платформе (платформа-хозяин) или виртуализирующая некоторую платформу и создающая на ней среды, изолирующие друг от друга программы и даже операционные системы.

Т.е. по-простому VM - это программная копия физического компьютера, которая выполняет ПО написанное под конкретную платформу, переводя скомпилированный байт-код в машинные операции.

Например Java VM исполняет байт-код java, созданный из исходного кода java-программы компилятором javac.

**Основные аспекты JVM:**

+ **Исполнение Java-кода** - JVM является средой исполнения, которая принимает байт-код Java и преобразует его в машинный код для конкретной платформы, на которой работает виртуальная машина.
+ **Кроссплатформенность** - Одним из ключевых преимуществ JVM является ее кроссплатформенность. Java-программы, скомпилированные в байт-код, могут выполняться на любой платформе, где установлена соответствующая JVM.
+ **Управление памятью** - JVM автоматически управляет памятью в Java-программах, включая выделение, освобождение и управление объектами в куче (heap).
+ **Сборка мусора** - JVM включает механизм сборщика мусора, который автоматически удаляет объекты, которые больше не используются приложением, освобождая память.
+ **Оптимизация кода** - JVM включает различные оптимизации кода, такие как JIT (Just-In-Time) компиляция, которые улучшают производительность выполнения Java-программ.
+ **Стек вызовов и куча** - JVM имеет стек вызовов для хранения локальных переменных и управления вызовами методов, а также кучу (heap) для хранения объектов.
+ **Иерархия классов загрузчиков** - JVM имеет иерархию классов загрузчиков (classloaders), которые загружают классы и ресурсы в виртуальную машину по требованию.

#### 2. К какому типу языка программирования относится Java? Что такое исходный код? Что такое байт-код?
**Java** — строго типизированный объектно-ориентированный язык программирования (ООП). Всё взаимодействие в нём происходит через объекты. Это в целом похоже на то, что творится в реальном мире: кот взаимодействует с хозяином, кассир — с покупателем, а клиент банка — со своим счётом. Все эти сущности описывают в коде и учат взаимодействовать друг с другом.

**Исходный код** — текст компьютерной программы на каком-либо языке программирования или языке разметки, который может быть прочтён человеком.

**Байт-код** — это стандартное промежуточное представление, в которое может быть переведена компьютерная программа автоматическими средствами. По сравнению с исходным кодом, удобным для создания и чтения человеком, байт-код — это компактное представление программы, уже прошедшей синтаксический и семантический анализ. В нём в явном виде закодированы типы, области видимости и другие конструкции. С технической точки зрения байт-код представляет собой машинно-независимый код низкого уровня, генерируемый транслятором из исходного кода.

#### 3. Из каких компонентов состоит Java (JDK, JRE, JVM)?

![img](https://github.com/shaporen/job4j_interviews/blob/main/resources/java_components.jpg)

Java Development Kit (JDK), Java Runtime Environment (JRE) и Java Virtual Machine (JVM) являются тремя основными компонентами платформы Java. Вот краткое описание каждого из них:

**Java Virtual Machine (JVM):**
   - JVM — это виртуальная машина, которая выполняет байт-код Java. Она является ключевым компонентом для выполнения Java-приложений и обеспечивает абстракцию от аппаратной платформы.
   - JVM интерпретирует байт-код Java на лету или JIT-компилирует его в машинный код для конкретной платформы.
   - JVM также управляет памятью, управлением потоками, загрузкой классов, безопасностью и другими аспектами выполнения Java-приложений.

**Java Runtime Environment (JRE):**
   - JRE представляет собой окружение выполнения для Java-приложений. Оно включает в себя виртуальную машину Java (JVM), необходимую для запуска байт-кода Java, а также базовые библиотеки Java API, необходимые для работы приложений.
   - JRE не содержит инструментов для разработки приложений, оно предназначено только для запуска уже скомпилированных Java-приложений.

**Java Development Kit (JDK):**
   - JDK — это полный комплект инструментов для разработки приложений на языке программирования Java. Он включает в себя компилятор Java (javac), который преобразует исходный код Java в байт-код, а также различные инструменты для создания, отладки и документирования Java-приложений.
   - В JDK также входят библиотеки Java API (Application Programming Interface), которые содержат набор классов и интерфейсов для работы с различными аспектами программирования, такими как ввод-вывод, работа с сетью, обработка строк, коллекции и т. д.

#### 4. Для чего используется JDK?

**JDK (Java Development Kit)** - это программное обеспечение, которое используется для разработки Java-приложений. Он содержит все необходимые инструменты для создания, компиляции, отладки и запуска Java-программ. 

JDK включает в себя:

- Java-компилятор (javac): переводит исходный код Java в байт-код Java.
- Java-архиватор (jar): создает JAR-файлы, которые являются архивами, содержащими классы Java, ресурсы и метаданные манифеста.
- Java-документация (javadoc): генерирует документацию API для классов и интерфейсов Java.
- Утилита java: запускает Java-программы, выполняя байт-код Java на виртуальной машине Java (JVM).
- Отладчик Java (jdb): интерактивный отладчик для Java-программ.
- Инструмент проверки кода (findbugs): находит возможные проблемы и ошибки в Java-коде.

JDK также включает библиотеки классов Java, которые предоставляют классы и интерфейсы для выполнения общих задач, таких как ввод-вывод, обработка строк и сетевое взаимодействие. JDK используется программистами Java для разработки, тестирования и развертывания Java-приложений. Он является необходимым инструментом для любого, кто хочет создавать программное обеспечение на языке Java.

В дополнение к перечисленным выше инструментам, JDK также включает в себя:

- Утилита keytool: управляет ключами и сертификатами, используемыми для цифровой подписи и шифрования.
- Утилита jarsigner: подписывает JAR-файлы цифровыми сертификатами.
- Утилита rmic: генерирует заглушки RMI (удаленного вызова метода).

#### 6. Для чего используется VM?

**Основные функции JVM:**

**1. Загрузка и проверка байт-кода:**
   * JVM загружает байт-код Java-программ (`.class` файлы), скомпилированный из исходного кода.
   * Затем JVM проверяет байт-код на корректность и безопасность, чтобы предотвратить потенциальные проблемы с кодом.

**2. Преобразование байт-кода:**
   * JVM преобразует байт-код в машинный код, который может выполняться на конкретной операционной системе и архитектуре компьютера. 
   * Этот процесс называется "интерпретацией", и он позволяет Java-программам работать независимо от платформы.

**3. Управление памятью:**
   * JVM управляет распределением памяти для Java-приложений, выделяя и освобождая память по мере необходимости. 
   * Это позволяет избежать утечек памяти и повышает стабильность программ.

**4. Обработка исключений:**
   + JVM перехватывает исключения, возникающие во время выполнения программы, и предоставляет возможность для их обработки.
   + Это помогает создавать более устойчивые и надежные приложения.

**5. Сбор мусора (Garbage Collection):**
   * JVM автоматически освобождает память, используемую объектами, которые больше не используются в программе. 
   * Это автоматическое управление памятью освобождает разработчиков от необходимости вручную выделять и освобождать память, упрощая разработку.

**В заключении:** JVM — это ключевая часть Java-платформы. Она позволяет Java-программам работать на разных компьютерах, управляет памятью и выполняет множество других задач, делая Java мощным и популярным языком программирования.

#### 7. Что такое переменная?

Переменная – это элементарный контейнер для хранения данных.

Как и в реальной жизни, контейнеры создают для хранения определенных вещей. Например: жидкости, горючих или сыпучих веществ и т.д. То есть у каждого контейнера есть определенный тип хранимых данных.

```java
ТИП_ДАННЫХ ИМЯ_ПЕРЕМЕННОЙ = ЗНАЧЕНИЕ_ПЕРЕМЕННОЙ;
```

#### 8. Расскажите про примитивные типы.

+ `byte` (целые числа, 1 байт или 8 бит, интервал значений [-128, 127], т.е. 2^7, т.к. отрицательные, ноль и положительные)
  Используется для хранения байтовой информации в памяти. 8 bits.
  ```java
  byte mem = 1;
  ```
+ `short` (целые числа, 2 байта или 16 бит, [-32768, 32767], т.е. 2^15)
  Целочисленный тип. 16 bits.
  ```java
  short size = 1;
  ```
+ `int` (целые числа, 4 байта, [-2147483648, 2147483647], т.е. 2^31)
  Целочисленный тип. 32 bits.
  ```java
  int length = 100500;
  ```
+ `long` (целые числа, 8 байт, [-9223372036854775808, 9223372036854775807], т.е. 2^63)
  Целочисленный тип. 64 bits.
  ```java
  long money = 900500;
  ```
+ `float` (вещественные числа, 4 байта, от 1.4e-45f до 3.4e+38f)
  Числа с плавающей точкой. 32 bits.
  ```java
  float size = 1.05F;
  ```
+ `double` (вещественные числа, 8 байт, от 4.9e-324 до 1.7e+308)
  Числа с плавающей точкой. 64 bits.
  ```java
  double size = 100500.99;
  ```
+ `char` (символ `Unicode`, 2 байта, [0, 65536], описывается одинарными ковычками)
  Символьный тип. 16 bits.
  ```java
  char exit = 'Y';
  ```
+ `boolean` (значение true/false, используется `int`, зависит от JVM)
  Описывают логический тип. 1 bit.
  ```java
  boolean exists = true;
  ```

Примитивы передаются по значению.

#### 10. Расскажите о конструкциях ветвления: if, else, switch, тернарное выражение.

**1. If-else:**

+ **if:**  Используется для выполнения блока кода, если условие истинно.
+ **else:**  Используется для выполнения блока кода, если условие ложно.
```java
int age = 25;
if (age >= 18) {
    System.out.println("Вам разрешено голосовать"); 
} else {
    System.out.println("Вам не разрешено голосовать");   
}
```

Можно использовать несколько вложенных блоков if-else.
```java
int grade = 85;
if (grade >= 90) {
    System.out.println("Отлично");
} else if (grade >= 80) {
    System.out.println("Хорошо");
} else if (grade >= 70) {
    System.out.println("Удовлетворительно");
} else {
    System.out.println("Неудовлетворительно");
}
```
**2. Switch:**

+ **switch:**  Используется для проверки значения переменной на соответствие нескольким случаям.
+ **case:**  Определяет конкретный случай, который нужно проверить.
+ **default:**  Используется для выполнения блока кода, если ни один из случаев не соответствует.
```java
int day = 3;

switch (day) {
    case 1:
        System.out.println("Понедельник");
        break;
    case 2:
        System.out.println("Вторник");
        break;
    case 3:
        System.out.println("Среда");
        break;
    default:
        System.out.println("Неизвестный день недели");
}
```

**3. Тернарное выражение**

Это короткий и компактный вариант записи if-else
```java
условие ? выражение1 : выражение2
```

**Как работает:**
+ **Условие:**  Оценивается как `true` или `false`.  
+ **Выражение1:** Выполняется и возвращает результат, если условие `true`. 
+ **Выражение2:** Выполняется и возвращает результат, если условие `false`.
```java
int age = 25;
String message = (age >= 18) ? "Вам разрешено голосовать" : "Вам не разрешено голосовать";
System.out.println(message); //Вывод: Вам разрешено голосовать
```

#### 12. Расскажите про отличия || и &&.

Логические операторы || и && в языке программирования Java используются для выполнения логических операций над булевыми значениями. Вот их основные отличия:
1. Оператор || (логическое "или") возвращает true, если хотя бы один из операндов равен true. То есть, если хотя бы одно из условий выполняется, результат будет true. Пример:
```
boolean a = true;
boolean b = false;
if (a || b) {
    System.out.println("Один из операндов равен true");
}
```
2. Оператор && (логическое "и") возвращает true, если оба операнда равны true. То есть, оба условия должны быть выполнены, чтобы результат был true. Пример:
```
boolean a = true;
boolean b = true;
if (a && b) {
    System.out.println("Оба операнда равны true");
}
```
3. Оператор || исключает вычисление второго операнда, если первый равен true. Таким образом, если первый операнд равен true, второй операнд не проверяется, так как результат будет true в любом случае. Это называется "ленивая оценка" (short-circuit evaluation). Пример:
```
boolean a = true;
boolean b = false;
if (a || b) {
    System.out.println("Этот текст будет выведен, несмотря на значение b");
}
```
4. Оператор && также использует ленивую оценку и исключает вычисление второго операнда, если первый равен false. Таким образом, если первый операнд равен false, второй операнд не проверяется, так как результат будет false в любом случае. Пример:
```
boolean a = false;
boolean b = true;
if (a && b) {
    System.out.println("Этот текст не будет выведен из-за значения a");
}
```
Это основные отличия между логическими операторами || и &&.

#### 13. Что такое тернарное условие?

Тернарное условие в Java - это оператор, который позволяет укороченное написание условия if-else. Он имеет следующий синтаксис:
```java
(условие) ? выражение1 : выражение2;
```
Если условие истинно, то будет выполнено выражение1, если ложно - выражение2.

Пример использования тернарного условия:
```java
int x = 10;
String result = (x > 5) ? "Больше 5" : "Меньше или равно 5";
System.out.println(result); // вывод: Больше 5
```

#### 14. Что такое циклы и для чего они используются?

Циклы в программировании - это структуры, позволяющие выполнять повторяющиеся действия определенное количество раз или до выполнения определенного условия. Они используются для автоматизации и оптимизации работы с повторяющимися задачами, такими как обход элементов массива, чтение данных из файлов, выполнение однотипных операций над данными и другие задачи. В зависимости от условий циклы могут выполняться разное количество раз или даже вовсе не выполняться. Использование циклов позволяет писать более компактный и эффективный код.

#### 15. Для чего используется цикл for?

Цикл for используется в Java для выполнения повторяющихся операций некоторое количество раз. Он имеет более компактный синтаксис, чем цикл while, и часто используется в случаях, когда известно заранее, сколько раз нужно выполнить определенные действия. Структура цикла for в Java выглядит следующим образом:
```java
for (инициализация; условие; инкремент) {
    // код, который нужно выполнить
}
```
Где:
- инициализация: задание начального значения переменной управления циклом
- условие: логическое выражение, которое определяет, когда нужно завершить цикл
- инкремент: операция, которая выполняется после каждой итерации цикла

Цикл for в Java позволяет более точно контролировать количество итераций и обеспечивает удобный способ для обработки массивов и коллекций.
```java
public class Main {
    public static void main(String[] args) {
        for (int i = 1; i <= 10; i++) {
            System.out.println("Iteration " + i);
        }
    }
}
```
Этот код будет выводить строку "Iteration " с номером текущей итерации от 1 до 10.

#### 16. Для чего используется цикл foreach?

Цикл foreach в Java используется для удобного перебора элементов в массивах, списке или других коллекциях. Он позволяет пройтись по каждому элементу коллекции без необходимости использования индексов, что делает код более читаемым и компактным. Данный цикл является удобной альтернативой традиционному циклу for и позволяет более эффективно работать с элементами коллекции.

Пример использования цикла foreach для итерации по массиву:
```java
int[] numbers = {1, 2, 3, 4, 5};

for (int number : numbers) {
    System.out.println(number);
}
```

Пример использования цикла foreach для итерации по коллекции (например, списку ArrayList):
```java
List<String> names = new ArrayList<>();
names.add("Alice");
names.add("Bob");
names.add("Charlie");

for (String name : names) {
    System.out.println(name);
}
```

#### 17. Для чего используется цикл while?

Цикл while в Java используется для повторения выполнения определенного блока кода до тех пор, пока условие, указанное внутри цикла, остается истинным. Это позволяет повторять определенные действия необходимое количество раз или до выполнения определенного условия. 

Например, цикл while может использоваться для обхода элементов массива, выполнения итераций в игровом цикле, проверки условий ввода пользователя и так далее. 

Синтаксис цикла while в Java выглядит следующим образом:

```java
while (условие) {
    // блок кода, который будет выполняться, пока условие равно true
}
```
Пример:
```java
public class WhileLoopExample {
    public static void main(String[] args) {
        int i = 0;
        while(i < 5) {
            System.out.println("Value of i: " + i);
            i++;
        }
    }
}
```
Данный цикл будет выводить значения от 0 до 4.

#### 18. Для чего используется цикл do while?

Цикл do while в Java используется для выполнения определенного блока кода, пока указанное условие истинно. В отличие от цикла while, в цикле do while блок кода выполняется хотя бы один раз, даже если условие не истинно.

Пример кода с использованием цикла do-while в Java:
```java
public class DoWhileExample {
    public static void main(String[] args) {
        int i = 1;
        
        do {
            System.out.println("Текущее значение i: " + i);
            i++;
        } while (i <= 5);
    }
}
```

В этом примере блок кода внутри цикла do будет выполняться до тех пор, пока переменная i меньше или равна 5. Даже если i изначально равно 10, блок кода все равно выполнится хотя бы один раз.

#### 19. Что такое массив?

Массив в Java — это структура данных, в которой хранятся элементы одного типа.

Его можно представить как набор пронумерованных ячеек, в каждую из которых можно поместить какие-то данные (один элемент данных в одну ячейку). Доступ к конкретной ячейке осуществляется через её номер. Номер элемента в массиве также называют индексом.

В Java массив однороден, то есть во всех его ячейках будут храниться элементы одного типа. Например, массив целых чисел содержит только целые числа (например, типа int), массив строк — только строки, массив из элементов созданного класса Dog будет содержать только объекты Dog.

#### 20. Как создать массив?

Для создания массива в Java используется ключевое слово new.

Пример создания массива:
```java
int[] array = new int[10];
```
Оператор new выделяет в памяти пространство для хранения десяти элементов типа int. После этого он возвращает ссылку, которая будет храниться в стеке, в начало области памяти, предназначенной под массив. Ссылка перемещается в переменную array, которая обладает таким же типом, как и массив.

После создания массива его размер (длину) изменить не получится.

int[] array = {1, 2, 3, 4, 5};
for (int i = 0; i < array.length; i++) {

int[] array = {1, 2, 3, 4, 5};

int[] array = {1, 2, 3, 4, 5};

Integer[] array = {1, 2, 3, 4, 5};
List<Integer> list = Arrays.asList(array);

#### 23. Как можно найти элемент в массиве?

1. Использовать цикл for для перебора всех элементов массива и проверки на равенство заданному значению.
Пример:
```java
int[] arr = {1, 2, 3, 4, 5};
int target = 3;
for (int i = 0; i < arr.length; i++) {
    if (arr[i] == target) {
        System.out.println("Элемент найден на позиции " + i);
    }
}
```

2. Использовать метод Arrays.binarySearch() для поиска элемента в отсортированном массиве.
Пример:
```java
import java.util.Arrays;

int[] arr = {1, 2, 3, 4, 5};
int target = 3;
int index = Arrays.binarySearch(arr, target);
if (index >= 0) {
    System.out.println("Элемент найден на позиции " + index);
}
```

3. Использовать метод Arrays.asList() для преобразования массива в список и использования метода indexOf() для поиска элемента.
Пример:
```java
import java.util.Arrays;

int[] arr = {1, 2, 3, 4, 5};
int target = 3;
int index = Arrays.asList(arr).indexOf(target);
if (index >= 0) {
    System.out.println("Элемент найден на позиции " + index);
}
```

#### 24. Что будет, если записывать элемент по индексу -1?

В Java массивы индексируются с 0, поэтому записывать элемент по индексу -1 приведет к ошибке. Попытка обратиться к элементу по индексу -1 вызовет `ArrayIndexOutOfBoundsException`, поскольку в массиве нет элемента с отрицательным индексом.
