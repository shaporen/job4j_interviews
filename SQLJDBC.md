#### 1. Что такое SQL?

SQL - это декларативный язык программирования, который используется для создания, модификации и управления данными в реляционной БД, управляемой системой управления базами данных (СУБД). С помощью SQL-запросов можно добавлять, изменять, получать или удалять данные из БД.

#### 2. Что такое DML и DDL?

**DML (Data Manipulation Language)** - язык манипулирования данными. Используется для изменения данных в уже существующих таблицах.

**_Операторы DML:_**

* **INSERT:** вставка новых строк в таблицу.
* **UPDATE:** изменение данных в существующих строках.
* **DELETE:** удаление строк из таблицы.
* **SELECT:** выборка данных из таблиц (хотя технически SELECT относится к DQL - Data Query Language, его часто относят к DML)

**DDL (Data Definition Language)** - язык определения данных. Используется для определения структуры базы данных.

**_Операторы DDL:_**

* **CREATE:** создание новых таблиц, баз данных, индексов, представлений и других объектов.
* **ALTER:** изменение структуры существующих объектов (добавленные колонки, изменение типов данных, удаление столбцов и т.д.).
* **DROP:** удаление таблиц, баз данных, индексов и других объектов.
* **TRUNCATE:** очистка данных в таблице, оставляя структуру.

**DQL (Data Query Language)** - это язык запросов к данным, используемый для извлечения информации из базы данных. Он является частью SQL (Structured Query Language - структурированный язык запросов) и позволяет получить данные из таблиц, а также выполнить некоторые операции, не изменяющие данные.

**_Операторы DQL:_**

* **SELECT:** основной оператор для выборки данных. Он позволяет указать, какие столбцы и строки нужно выбрать.
* **FROM:** определяет таблицу или таблицы, из которых нужно извлечь данные.
* **WHERE:** указывает условия для выборки данных.
* **ORDER BY:** определяет порядок сортировки результата.
* **GROUP BY:** группирует строки результата по заданному столбцу.
* **HAVING:** указывает условия для группировки данных.
* **JOIN:** объединяет данные из нескольких таблиц.

#### 3. Что такое первичный ключ?

**Первичный ключ (Primary Key) в базе данных** - это один или несколько столбцов в таблице, которые однозначно идентифицируют каждую строку в этой таблице. Первичный ключ, состоящий из нескольких полей, называется _составным ключом_.

Основные характеристики первичного ключа:

* Уникальность:  значение первичного ключа должно быть уникальным для каждой строки в таблице. 
* Неизменяемость:  значение первичного ключа, как правило, не должно изменяться после создания строки.
* Непустота:  значение первичного ключа не должно быть пустым (NULL).

Зачем нужен первичный ключ:

* Идентификация:  обеспечивает однозначную идентификацию каждой записи в таблице.
* Целостность данных:  помогает поддерживать целостность данных, предотвращая дублирование записей.
* Связь между таблицами:  используется для создания связей между таблицами (иностранные ключи).
* Поиск:  ускоряет поиск данных, так как позволяет быстро найти нужную строку по ее уникальному идентификатору.

Примеры первичных ключей:

* ID пользователя:  в таблице users столбец id может быть первичным ключом, который представляет собой уникальный номер для каждого пользователя.
* Номер заказа:  в таблице orders столбец order_id может быть первичным ключом, который представляет собой уникальный номер для каждого заказа.
* Номер продукта:  в таблице products столбец product_id может быть первичным ключом, который представляет собой уникальный номер для каждого продукта.

Важные моменты:

*  В таблице может быть только один первичный ключ.
*  Часто в качестве первичного ключа используется столбец с типом данных INT или BIGINT, который автоматически генерируется (AUTO_INCREMENT).
*  В некоторых случаях первичный ключ может состоять из нескольких столбцов. 

Пример создания таблицы с первичным ключом:
```java
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(255),
  email VARCHAR(255)
);
```
В этом примере id - это первичный ключ таблицы users, который является уникальным для каждого пользователя.

#### 4. Что такое внешний ключ?

**Внешний ключ (Foreign Key) в базе данных** - это столбец (или комбинация столбцов) в одной таблице, который ссылается на первичный ключ другой таблицы (или несколько первичных ключей разных таблиц). 

Основные функции внешнего ключа:

* Создание связей между таблицами: внешние ключи позволяют устанавливать связи между различными таблицами в базе данных, создавая логические отношения между записями.
* Обеспечение целостности данных: внешние ключи гарантируют, что данные в связанных таблицах остаются согласованными. Например, если в таблице "заказы" есть строка с номером продукта, который отсутствует в таблице "продукты", то база данных не позволит сохранить эту запись.
* Упрощение запросов: внешние ключи упрощают выполнение запросов, позволяя получать данные из нескольких таблиц одновременно.
